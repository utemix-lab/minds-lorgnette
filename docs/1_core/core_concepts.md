# Концепция: Configurable Core + Drivers

## 1. Конфигурируемое ядро (Configurable Core)

Это центральный координатор. Его единственная ответственность:
*   Загружать конфигурацию.
*   Инициализировать указанные Драйверы.
*   Обеспечивать коммуникацию между ними через единую **шину событий (Event Bus)**.
*   Предоставлять внешний API для работы с системой.

**Ядро не содержит предметной логики.** Это "скелет", на который наращиваются "мышцы" (Драйверы).

## 2. Драйверы (Drivers)

Драйвер — это независимый модуль, реализующий один из аспектов работы со знаниями.

| Тип драйвера | Отвечает на вопрос | Примеры реализаций |
| :--- | :--- | :--- |
| **`Ontology Driver`** | "Во что превращается моя мысль?" | Иерархия тегов, графовая схема, динамические категории. |
| **`Semantics Driver`** | "Как система понимает смысл?" | Ключевые слова (TF-IDF), векторные эмбеддинги (OpenAI, BERT), вызов LLM. |
| **`Connectivity Driver`** | "Как это связано с другим?" | Обратные ссылки, автоматические ассоциации, граф знаний (GraphRAG). |
| **`Interface Driver`** | "Как с системой взаимодействовать?" | REST API, WebSocket для агентов, файловый экспорт, UI-виджет. |
| **`Storage Driver`** | "Где это хранится?" | Локальные файлы (JSON, SQLite), векторная БД (Chroma), графовая БД (Neo4j). |

## 3. Принцип работы

1.  **Конфигурация**: Пользователь описывает стек в `mind-stack.yaml`.
2.  **Загрузка**: Ядро читает конфиг, загружает нужные драйверы (как плагины).
3.  **Инициализация**: Ядро передаёт каждому драйверу его настройки и подключает к шине событий.
4.  **Работа**: При поступлении новой информации (например, текста) ядро инициирует пайплайн обработки через последовательность драйверов.
