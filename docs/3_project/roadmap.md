# Дорожная карта (Concept Phase)

Цель этой карты — превратить концепцию в работающий open-source проект и сообщество.

## Этап 0: Закрепление идеи и документация (Текущий)
*   [x] Сформулировать базовое видение и архитектуру.
*   [x] Создать публичный репозиторий с документацией (этот репо!).
*   [ ] Наполнить раздел `/examples` первыми конфигурационными файлами.
*   [ ] Написать статью/пост, представляющий проект сообществу.

## Этап 1: Доказательство концепции (Proof of Concept)
*   **Цель**: Создать минимальное работающее ядро и 2-3 драйвера.
*   **Задачи**:
    *   Разработать спецификацию API для Драйверов.
    *   Реализовать ядро, способное загружать драйверы из конфига.
    *   Создать базовые драйверы: `YamlOntologyDriver`, `SentenceTransformersSemanticsDriver`, `FileStorageDriver`.
    *   Показать рабочий пайплайн на примере из `/examples`.

## Этап 2: Альфа-релиз для ранних пользователей
*   **Цель**: Привлечь первых разработчиков и получить фидбэк.
*   **Задачи**:
    *   Публичный релиз пакета на PyPI (или другом менеджере).
    *   Подробные туториалы и документация по созданию своих драйверов.
    *   Интеграция с популярными инструментами (напр., записные книжки Jupyter, Obsidian через плагин).

## Этап 3: Формирование экосистемы
*   **Цель**: Стать точкой сборки для семантических инструментов.
*   **Задачи**:
    *   Создать реестр (registry) сторонних драйверов.
    *   Предоставить адаптеры для популярных LLM API и векторных БД.
    *   Запустить регулярные воркшопы и хакатоны.

## Как можно помочь сейчас?

1.  **Обсуждение**: Поделитесь своими use-case в [Discussions](https://github.com/utemix-lab/minds-lorgnette/discussions). Что бы вы хотели собрать?
2.  **Примеры**: Создайте конфигурацию для своей задачи в `/examples`.
3.  **Распространение**: Расскажите о проекте коллегам, которым близка тема гибких инструментов для работы с информацией.
